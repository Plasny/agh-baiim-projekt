Zadanie 1. (GET)

Ktoś zaimplementował zmianę ustawień konta za pomocą zapytań GET, jak można to wykorzystać?

Przygotuj stronę, która po załadowaniu (bez udziału użytkownika) automatycznie zmieni hasło na stronie ustawień aplikacji (http://localhost:8081/task1) na "abc12345".

Najpierw zobacz jak działa ta strona: /task1

Zadanie dodatkowe

Spróbuj to zrobić w taki sposób żeby użytkownik nie zobaczył, że hasło zostało zmienione. Żeby strona nie została przekierowana.

Podpowiedź 1 (od czego zacząć) Zobacz jak wygląda ruch sieciowy w zakładce Network narzędzi developerskich przeglądarki podczas zmiany hasła.
Zakładkę tą można otworzyć za pomocą skrótu klawiaturowego F12 lub Ctrl+Shift+I (Cmd+Option+I na Macu).
Podpowiedź 2 (co warto przeczytać)
  • Window load event (MDN)
  • Window location (MDN)
  • Window fetch (MDN)
  • HTML img element (MDN)
Na jakie sposoby można to zrobić? - Za pomocą przekierowania (window.location)
- Za pomocą fetch API
- Za pomocą elementów jak <img> lub <script>
